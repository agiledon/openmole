## 1. 目的

在 **重构编码之前**，校验 **BDR 文档链** 一致性：

- **宪法**（[constitution.md](./constitution.md)）— 最高规范是否被遵守；  
- **元规约**（[specification.md](./specification.md)）— 条目格式与门禁；  
- **坏味道规约**（`**[badsmells.md](./badsmells.md)`**）— 具体条目是否可验收、是否与 SDD 冲突；  
- **任务**（[tasks.md](./tasks.md)）— 是否 **完全源自 `badsmells.md`**、依赖是否无环、DoD 是否仍有效。

**冲突解决**：**先改文档**（宪法 / 元规约 / `**badsmells.md`** / **tasks.md** / 本文件），**再改代码**。

**提交版本**：**分析** 环节每次 **升版本文件** 或 **在 §5 修订历史中新增一行**（含完成 §2 差分并更新 `tasks.md`）时，须按 [specification.md §7](./specification.md#7-修订历史表与提交版本门禁) 在 **修订历史** 中填写 **提交版本**（`git rev-parse HEAD`）。

---

## 2. `badsmells.md` 变更后的强制差分（核心）

当 `**badsmells.md` 的版本号或实质性内容** 发生变化时，**必须** 执行本节检查并 **更新 [tasks.md](./tasks.md)**（见 [constitution.md §2.4](./constitution.md#24-分析analyze环节与文档迭代)）：


| 步骤  | 动作                                                                                                |
| --- | ------------------------------------------------------------------------------------------------- |
| A   | 列出 `**badsmells.md` 当前** 全部条目 ID（或编号）                                                             |
| B   | 列出 **tasks.md 当前** 各任务引用的 BS-ID / 章节                                                              |
| C   | **新增条目** → 在 **tasks.md** 增补任务或并入现有任务                                                             |
| D   | **删除或合并条目** → 删除或改写 **tasks.md** 中孤儿/过时任务                                                         |
| E   | **条目验收标准变更** → 同步对应任务的 **DoD**                                                                    |
| F   | 将本节结论 **摘要写入本文 §3**（或修订历史）；**本次文档修订** 在 §5 **修订历史** 新增行中的 **提交版本** 填 `**git rev-parse HEAD`** 的输出 |


**未完成差分并更新任务前**，不得将 `**badsmells.md` 的新增要求** 直接当作已批准任务实施。

## 3. 三角 / 多角检查清单（评审用）


| 检查项                     | 说明                                                                                              |
| ----------------------- | ----------------------------------------------------------------------------------------------- |
| 宪法 ↔ 元规约                | [specification.md](./specification.md) 是否与 [constitution.md §3](./constitution.md#3-重构第一性原则) 对齐 |
| 元规约 ↔ `badsmells.md`    | [specification.md §4](./specification.md#4-清单条目格式写入-badsmellsmd-的规约要求) 必填字段是否齐全；**[SDD]** 是否标注  |
| `**badsmells.md` ↔ 任务** | 每个 `B-Txx` 是否引用 `**badsmells.md`** 条目；是否存在 **无任务覆盖** 的高优先级坏味道或 **无条目支撑** 的任务                    |
| 任务 ↔ 宪法                 | DoD 是否含 **测试绿 + 用户确认**；是否跳过 [constitution §4](./constitution.md#4-重构的标准步骤)                      |
| BDR ↔ SDD               | 对外行为变更是否已在 SDD 立项                                                                               |

## 4. 已知结论与待决事项（维护者填写）

- **[badsmells.md](./badsmells.md)** 为坏味道 **唯一正式规约**（具体条目与统计）。  
- `**badsmells.md` 迭代 → 本节差分 → 更新 tasks.md** 为强制流程。

### 4.2 待决事项

- 维护者确认 [constitution.md](./constitution.md) **v1.4**（含 **§3 一致**、**健壮/安全**、**§7 提交版本** 门禁）。  
- 填充 [badsmells.md](./badsmells.md) 条目并执行 **§2 差分**，更新 [tasks.md](./tasks.md)（**v0.2.0 / 2026-03-31**；后续 **v0.2.1 / v0.3.1** 对齐 **提交版本** 门禁）。  
- 按 [tasks.md](./tasks.md) 逐项重构并在每任务后 **用户确认**。
